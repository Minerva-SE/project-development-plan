\section{范围与交付物}

\subsection{项目范围}

本节依据需求分析报告中定义的系统定位与功能边界，明确本项目的包含范围与排除范围，作为需求变更控制和验收判定的基准。

\subsubsection{包含范围}

本项目的开发与交付范围包括以下功能模块和工程产物：

\begin{enumerate}
  \item \textbf{用户与权限管理模块}——用户注册（含验证码）、登录登出（JWT Token 认证）、角色区分（被试者 / 实验员 / 管理员）、账号审批与状态管理（待审核 / 已激活 / 已驳回 / 已禁用）、个人信息查看与修改、密码修改。
  \item \textbf{实验项目管理模块}——实验项目的创建、编辑、查看、列表展示和状态流转（草稿 $\to$ 进行中 $\to$ 已结束 $\to$ 已归档），实验元数据（名称、编号、描述、用途类型、时间范围、被试者关联）的结构化管理。
  \item \textbf{EEG 数据管理模块}——EEG 原始数据文件的上传（支持 CSV、EDF 等格式）、元数据录入（采样率、通道数、通道名称、采集时间、标签等）、文件格式与大小校验、数据列表与详情展示、数据状态流转（已上传 $\to$ 处理中 $\to$ 待审核 $\to$ 已通过 / 已拒绝 $\to$ 已冻结 $\to$ 已归档）、标签管理与分类检索、软删除。
  \item \textbf{自动解析与质控模块}——文件格式解析、采样率一致性校验、通道完整性检查、缺失值比例统计、异常值检测、综合质控评分（通过 / 警告 / 不通过）与处理建议生成、质控报告展示与下载。
  \item \textbf{伦理审核与授权模块}——知情同意书模板的创建、版本管理和生效状态控制（同时仅一个 Active 版本）；被试者在线阅读与电子化签署知情同意书（支持撤回）；实验员提交数据使用申请（用途、访问范围、导出格式）；管理员审批数据使用申请并指定授权范围与有效期限；被试者查看授权记录并撤回授权。
  \item \textbf{数据分析模块}——时域波形可视化、统计摘要（均值/方差/峰值）、FFT 频谱分析、频段能量计算（$\alpha$、$\beta$ 等波段）、AI 状态识别；分析任务的创建、状态追踪和结果展示。
  \item \textbf{通知与 Dashboard 模块}——角色化首页仪表盘（按被试者 / 实验员 / 管理员分别聚合关键指标和待办事项）；站内通知推送（审核结果、授权申请、撤回通知、系统公告）；通知的已读 / 未读状态区分。
  \item \textbf{系统配置与审计模块}——管理员维护系统运行参数（上传大小上限、允许文件格式、Token 有效期、验证码有效期、日志保留时长等）；操作日志的自动记录与按条件检索（操作人、操作类型、对象类型、时间范围）。
  \item \textbf{前端界面}——基于 React + TypeScript 的响应式 Web 界面，覆盖上述所有功能模块的页面与交互，支持浅色/深色主题切换。
  \item \textbf{后端 API}——基于 Python/Django 的 RESTful API 服务，覆盖用户认证、实验管理、数据管理、质控报告、伦理审核、数据分析、通知、日志和系统配置等全部接口。
  \item \textbf{数据库设计}——包含用户、实验、EEG 数据、质控报告、处理结果、知情同意、访问申请、授权、通知、操作日志、分析结果和系统配置等核心数据表的完整设计与实现。
  \item \textbf{工程文档}——需求分析报告、项目开发计划书、系统设计文档、测试报告、项目总结和部署说明。
\end{enumerate}

\subsubsection{排除范围}

以下内容明确不属于本项目的开发与交付范围：

\begin{enumerate}
  \item \textbf{高级信号源定位分析}——不涉及 LORETA、Beamformer、sLORETA 等 EEG 源定位算法。
  \item \textbf{临床诊断辅助}——不提供面向临床诊断的自动化判读、疾病分类或辅助决策功能，系统的分析结果仅供教学和基础科研参考。
  \item \textbf{真实伦理委员会（IRB）审批}——系统内的“伦理审核”为教学模拟流程，不替代真实机构伦理委员会的法律审查职能，不产生具有法律效力的伦理审批文件。
  \item \textbf{大规模联邦计算}——不支持跨机构分布式数据集的联邦学习或联合分析。
  \item \textbf{EEG 硬件驱动与实时采集}——不包含 EEG 硬件设备的驱动开发、实时数据流采集或设备控制功能；系统仅管理已采集完成的离线数据文件。
  \item \textbf{移动端原生应用}——不开发 iOS/Android 原生 App，系统以 Web 浏览器为主要访问终端。
  \item \textbf{多语言国际化}——当前仅支持简体中文界面，不实现 i18n 多语言切换。
  \item \textbf{支付与计费系统}——不涉及任何商业支付、订阅计费或许可证管理功能。
\end{enumerate}

\subsubsection{交付物说明}

\begin{itemize}
  \item \textbf{需求分析报告}：系统功能需求、数据流图、状态图、CRC 卡、类图、数据词典、UI 原型、验收标准和环境规定的完整描述，采用 LaTeX 编撰。
  \item \textbf{项目开发计划书}：项目目标、范围、工作分解、组织分工、进度安排、质量管理、风险管理和沟通计划的完整描述。
  \item \textbf{系统设计文档}：系统架构设计、数据库设计、接口设计和前端组件设计的详细说明。
  \item \textbf{测试报告}：单元测试、集成测试和验收测试的测试用例、执行结果和缺陷统计。
  \item \textbf{项目总结报告}：项目执行过程回顾、目标达成情况评估、经验教训总结和后续改进建议。
  \item \textbf{部署说明}：系统运行环境要求、依赖安装步骤、配置文件说明和启动流程。
  \item \textbf{源代码}：前端（React + TypeScript）和后端（Python/Django）完整源代码，含构建配置、依赖声明和 README 说明，托管于 GitHub Organization 仓库。
  \item \textbf{演示材料}：答辩幻灯片（PPT）和核心流程演示视频，覆盖从用户注册到 AI 分析的全链路业务场景。
\end{itemize}
